Options Detail

Option 1: Hand-Crafted C99

Approach: Analyze PyTorch model → export weights via Python → implement all ShuffleNet V2 ops (conv2d, depthwise conv, BN, ReLU, channel shuffle, global avg pool, FC) in C99.

Strengths: Smallest code (1,660 lines), fully human-readable, maximum optimization control, int8 quantization support, CMSIS-NN compatible, BN fusion possible, ping-pong buffer memory management.

Weaknesses: Manual effort (~2 days), error-prone for complex architectures, requires deep understanding of ShuffleNet V2 internals.

Unique feature: Only option supporting int8 quantization (2.2 MB weights vs 8.7 MB float32).

Key files: shufflenet_v2.c (1,215 lines), shufflenet_v2.h (362 lines), main.c (445 lines), export_weights.py

Best for: Production deployment requiring maximum performance and minimum footprint.

Option 2: MATLAB Coder Support Package for PyTorch (R2026a)

Approach: loadPyTorchExportedProgram loads .pt2 model → MLIR/TOSA lowering → Embedded Coder generates C for STM32.

Strengths: Fully automated end-to-end, native STM32 hardware support, single monolithic C file (62,977 lines), numerically equivalent to PyTorch.

Weaknesses: New R2026a feature (less mature), requires Python environment with exact PyTorch version, large generated code.

Unique feature: Direct coder.hardware('STM32F746G-Discovery') support with Embedded Coder.

Key files: mInvoke_shufflenet.c (62,977 lines), generate_code_pytorch_coder.m

Best for: Rapid prototyping with guaranteed PyTorch numerical equivalence.

Option 3: importNetworkFromPyTorch → dlnetwork → MATLAB Coder

Approach: Import .pt2 model as dlnetwork using Deep Learning Toolbox → analyze for codegen → generate C with MATLAB Coder + Embedded Coder.

Strengths: Mature workflow (since R2022b), inspectable dlnetwork graph, ARM Compute Library support (ShuffleNet explicitly listed), analyzeNetworkForCodegen validation.

Weaknesses: May need custom layer fixes (chunk, view, channel_shuffle ops), requires expandLayers for codegen.

Unique feature: ShuffleNet listed as supported for ARM Compute Library — optimized ARM kernels available.

Key files: import_and_generate.m, predict_shufflenet.m, +shufflenet_exported/ (custom layers)

Best for: Recommended path — best balance of automation, optimization, and maturity.

Option 4: PyTorch → ONNX → importNetworkFromONNX → MATLAB Coder

Approach: Export to ONNX (opset 17, legacy exporter) → import as dlnetwork → generate C. R2026a: auto-generated custom layers support codegen.

Strengths: Most portable format (ONNX), R2026a custom layer codegen confirmed working, broadest ecosystem support, GenerateCoderFiles=true generates +coder package.

Weaknesses: Two-step conversion (extra ONNX intermediate), more custom layers generated (Reshape, Slice, Gather, Squeeze ops), largest code output.

Unique feature: R2026a first release where auto-generated ONNX custom layers support C code generation out of the box.

Key files: export_to_onnx.py, import_onnx_and_generate.m, +shufflenet_v2/+coder/ (codegen layers)

Best for: Fallback when PyTorch import has issues, or when ONNX is the only available format.

Issues Encountered & Fixes

Option 1: Hand-Crafted C

  • Channel shuffle complexity — ShuffleNet V2's channel shuffle (reshape+transpose+flatten) must be implemented efficiently without temporary allocation.
    Fix: In-place interleave with ping-pong buffer swap, avoiding extra memcpy.
  • Weight layout mismatch — PyTorch stores conv weights as [out_ch, in_ch, kH, kW], C implementation needs careful indexing.
    Fix: export_weights.py preserves PyTorch layout; C code indexes with matching stride calculations.

Option 2: PyTorch Coder (MLIR)

  • PyTorch version constraint — loadPyTorchExportedProgram requires specific PyTorch version matching the export.
    Fix: Use matching PyTorch version in Python venv, or re-export with compatible version.
  • Large monolithic output — Single 62,977-line C file with embedded weight constants.
    Impact: Compilation may be slow; consider splitting for large-scale projects.

Option 3: importNetworkFromPyTorch

  • Custom layers for ShuffleNet ops — channel_shuffle (view+transpose), chunk, and mean operations generate custom layers in +shufflenet_exported/.
    Fix: Added %#codegen pragmas to all custom layer files with fix_codegen_pragmas.m.
  • NetworkLayer containers — Nested module structure may need expandLayers() for code generation compatibility.

Option 4: ONNX Import

  • External data format — PyTorch dynamo ONNX exporter creates .onnx.data external weight file; MATLAB cannot import.
    Fix: Use legacy ONNX exporter (dynamo=False, opset 17).
  • Custom layer codegen — 17 auto-generated custom layers (Shape, Slice, Reshape, Gather, Squeeze, ReduceMean) with separate +coder package.
    Confirmed: R2026a GenerateCoderFiles=true creates codegen-compatible +coder package automatically.
  • Shape-dependent layers — ONNX Shape/Slice/Gather ops for dynamic reshape at channel shuffle generate multiple custom layers.
    Fix: Static input size eliminates runtime shape computation; codegen handles correctly.

Code Samples

Option 1: ShuffleNet V2 Channel Shuffle (C99)

/* Channel shuffle: interleave two halves of channels
 * Input:  C channels in CHW layout
 * Output: channels interleaved as [c0, c_half, c1, c_half+1, ...]
 * This is the key operation in ShuffleNet V2. */
static void channel_shuffle(const float *in, float *out,
                            int channels, int height, int width) {
    const int half = channels / 2;
    const int spatial = height * width;
    for (int c = 0; c < half; c++) {
        const float *src0 = in + c * spatial;
        const float *src1 = in + (half + c) * spatial;
        float *dst0 = out + (2 * c) * spatial;
        float *dst1 = out + (2 * c + 1) * spatial;
        memcpy(dst0, src0, spatial * sizeof(float));
        memcpy(dst1, src1, spatial * sizeof(float));
    }
}

Option 2: MATLAB Entry Point

function out = mInvoke_shufflenet(modelFile, input)
%#codegen
% Entry-point for PyTorch Coder support package code generation
persistent net;
if isempty(net)
    net = loadPyTorchExportedProgram(modelFile);
end
out = net.invoke(input);
end

Option 3: Import & Generate Workflow

% Step 1: Import PyTorch model as dlnetwork
net = importNetworkFromPyTorch('shufflenet_exported.pt2', ...
    PyTorchInputSizes=[1 3 224 224]);

% Step 2: Save for codegen
save('shufflenet_dlnetwork.mat', 'net');

% Step 3: Configure Embedded Coder
cfg = coder.config('lib', 'ecoder', true);
cfg.Hardware = coder.hardware('STM32F746G-Discovery');
cfg.DeepLearningConfig = coder.DeepLearningConfig('none');

% Step 4: Generate C code
inputType = coder.typeof(single(0), [224 224 3 1]);
codegen -config cfg predict_shufflenet -args {inputType}
